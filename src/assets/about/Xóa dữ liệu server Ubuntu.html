🧠 Tổng quan cache nằm ở đâu?

Trên Ubuntu/Debian, cache nằm ở mấy chỗ chính:

  1. OS (APT, temp)

  2. RAM cache

  3. Log

  4. User cache

  5. Docker (rất nhiều cache)

  6. App (Node, npm, pip…)

🧹 1. Xóa cache hệ điều hành (APT)

Bash
sudo apt clean
sudo apt autoclean

👉 Xóa:

  • file .deb tải về trong /var/cache/apt/archives

🧹 2. Xóa cache temp hệ thống

Bash
sudo rm -rf /tmp/*
sudo rm -rf /var/tmp/*

🧹 3. Xóa cache user

Bash
rm -rf ~/.cache/*

Hoặc full user:

Bash
sudo rm -rf /home/*/.cache

🧹 4. Xóa log (rất nặng nếu server chạy lâu)

Bash
sudo rm -rf /var/log/*

🧹 5. Clear RAM cache (Linux cache memory)

Bash
sudo sync
echo 3 | sudo tee /proc/sys/vm/drop_caches

👉 Giải thích:

  • 1 = page cache

  • 2 = dentries + inode

  • 3 = ALL

🐳 6. XÓA CACHE DOCKER (quan trọng nhất)

🔥 Dọn nhanh toàn bộ:

Bash
docker system prune -a --volumes -f

👉 Xóa:

  • container stop

  • image không dùng

  • network

  • volume (⚠️ DB cũng bay)

🧨 Dọn sâu hơn (manual full):

Bash
docker builder prune -a -f
docker image prune -a -f
docker container prune -f
docker volume prune -f
docker network prune -f

📁 Nếu muốn xoá tận gốc Docker cache:

Bash
sudo rm -rf /var/lib/docker/*

⚠️ Cực mạnh:

  • reset Docker về trắng

  • mất toàn bộ data

🧹 7. Cache của Node / npm (nếu bạn dùng React)

Bash
npm cache clean --force

🧹 8. Cache Python (nếu có)

Bash
pip cache purge

🧹 9. Journal logs (systemd)

Bash
sudo journalctl --vacuum-time=1d

👉 giữ lại log 1 ngày thôi

🚀 Combo lệnh “dọn sạch server” (gọn nhất)

Chạy 1 phát:

Bash
sudo apt clean && \
sudo rm -rf /tmp/* /var/tmp/* && \
sudo rm -rf /var/log/* && \
docker system prune -a --volumes -f && \
npm cache clean --force 2>/dev/null && \
pip cache purge 2>/dev/null
